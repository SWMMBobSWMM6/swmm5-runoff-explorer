🔗 Running SWMM Hydrology in ICM InfoWorks

      

        InfoWorks ICM can run EPA SWMM runoff natively within an
        InfoWorks network. The same three-surface nonlinear reservoir model —
        impervious without depression storage, impervious with depression storage, and pervious with
        infiltration — is available as a routing model type called "SWMM" in the
        ICM Runoff Surface editor. This page maps every SWMM5 parameter to its ICM InfoWorks equivalent.
      

    


    
    

      
Step-by-Step: Setting Up SWMM Runoff in ICM InfoWorks

      

        

          
1

          
Create 3 Runoff Surfaces

          
In the Runoff Surface editor, create three surfaces:

            • Imperv_noDS — Fixed runoff (coeff=1.0), Routing=SWMM

            • Imperv_wDS — Fixed runoff (coeff=1.0), Routing=SWMM

            • Pervious — Horton or Green-Ampt volume model, Routing=SWMM

            Set Manning's n in the Runoff routing value field for each.

            Set the area percentages for each surface to match your SWMM5 subcatchment's
            %Imperv and %Zero-Imperv splits.

In the Subcatchment editor, assign the Land Use to each subcatchment.
            Set the total area, slope, and subcatchment width.
            The width applies to all three surfaces in ICM (same as SWMM5).

        

        

          
4

          
Set Infiltration

          
On the pervious Runoff Surface, set the infiltration parameters:

            • Horton: f₀ (max rate), f∞ (min rate), k (decay)

            Set initial conditions in the Rainfall Event editor.

Configure Rainfall Event

          
Add a Rainfall Event with the appropriate profile.
            Set initial conditions for Horton moisture store or Green-Ampt moisture deficit.
            Match the subcatchment Rainfall Profile name to the event profile name.

        

        

          
6

          
Run Simulation

          
Create a Run with the rainfall event. ICM will use the SWMM nonlinear reservoir +
            Manning's overland flow equation (the same RK5 ODE solver) for each of the three surfaces
            on every subcatchment.

⚠️ Key Differences & Notes


      

        Three-Level Hierarchy: In SWMM5, surface parameters live directly on the subcatchment
        ([SUBAREAS] + [INFILTRATION] sections). In ICM InfoWorks, there's a three-level hierarchy:
        Subcatchment → Land Use → Runoff Surface. Multiple subcatchments can share the same
        Land Use, and multiple Land Uses can share the same Runoff Surface definitions.
      


      

        Width Field: In SWMM5, the width is set once on the subcatchment and applies to all
        three surfaces. In ICM InfoWorks, the width is also on the subcatchment, and the SWMM routing
        model uses it for all surfaces. The Manning's n is always the Runoff routing value
        field regardless of what routing type is displayed in the UI.
      


      

        Slope-Related vs Absolute Initial Loss: ICM allows both slope-related initial loss
        (recommended value 0.000071 m for impervious, matching Wallingford model) and absolute depth.
        SWMM5 always uses absolute depth for depression storage. Use Absolute type in ICM to
        match SWMM5 behavior exactly.
      


      

        Infiltration Initial Conditions: SWMM5 puts initial moisture/deficit in the [INFILTRATION]
        section. ICM requires these in the Rainfall Event editor → Initial Conditions tab.
        This is a common gotcha during conversion — if you miss it, Horton starts fully recovered
        and Green-Ampt starts fully saturated.
      


      

        Groundwater: ICM supports groundwater modeling but uses its own InfoWorks method rather
        than the SWMM5 two-zone groundwater equations. If your SWMM5 model has [GWF] sections, these
        need manual re-calibration in ICM.
      


      

        SWMM5 Import: ICM can directly import SWMM5 .INP files. The importer automatically
        creates the Runoff Surfaces, Land Uses, and Subcatchment assignments. It creates one Runoff
        Surface per unique parameter set and shares them across subcatchments.
      

    


    
    

      
💎 Ruby Script: Auto-Configure SWMM Runoff Surfaces in ICM

      
# Ruby script to set up SWMM hydrology in ICM InfoWorks
# Run from ICM UI Script → applies to current network

net = WSApplication.current_network

# --- Step 1: Define three SWMM Runoff Surfaces ---
net.transaction_begin

# Surface A1: Impervious, no depression storage
s1 = net.new_row_object('hw_runoff_surface')
s1['surface_id']            = 'SWMM_Imperv_noDS'
s1['routing_model']          = 'SWMM'
s1['runoff_volume_type']     = 'Fixed'
s1['fixed_runoff_coefficient'] = 1.0
s1['runoff_routing_value']   = 0.012   # Manning's n
s1['initial_loss_type']      = 'Absolute'
s1['initial_loss_value']     = 0.0     # No depression storage
s1.write

# Surface A2: Impervious, with depression storage
s2 = net.new_row_object('hw_runoff_surface')
s2['surface_id']            = 'SWMM_Imperv_wDS'
s2['routing_model']          = 'SWMM'
s2['runoff_volume_type']     = 'Fixed'
s2['fixed_runoff_coefficient'] = 1.0
s2['runoff_routing_value']   = 0.012   # Manning's n
s2['initial_loss_type']      = 'Absolute'
s2['initial_loss_value']     = 0.05    # 0.05 in depression storage
s2.write

# Surface A3: Pervious, Horton infiltration
s3 = net.new_row_object('hw_runoff_surface')
s3['surface_id']            = 'SWMM_Pervious'
s3['routing_model']          = 'SWMM'
s3['runoff_volume_type']     = 'Horton'
s3['runoff_routing_value']   = 0.15    # Manning's n for overland
s3['initial_loss_type']      = 'Absolute'
s3['initial_loss_value']     = 0.20    # 0.20 in depression storage
s3['horton_max_rate']        = 3.0     # f0 (in/hr)
s3['horton_min_rate']        = 0.5     # fc (in/hr)
s3['horton_decay']           = 4.0     # k  (1/hr)
s3.write

net.transaction_commit
puts "Created 3 SWMM runoff surfaces"

# --- Step 2: Create Land Use referencing all 3 surfaces ---
net.transaction_begin

lu = net.new_row_object('hw_land_use')
lu['land_use_id'] = 'SWMM_Mixed'
# Set surface areas (example: 50% imperv, 25% of that has no DS)
# A1 = 50% × 25% = 12.5%,  A2 = 50% × 75% = 37.5%,  A3 = 50%
lu.write

net.transaction_commit

# --- Step 3: Assign to subcatchments ---
net.transaction_begin

net.row_objects('hw_subcatchment').each do |sc|
  sc['land_use_id'] = 'SWMM_Mixed'
  # Width comes from SWMM5 subcatchment width
  # sc['width'] = 500  # ft — set per subcatchment
  sc.write
end

net.transaction_commit
puts "Assigned SWMM land use to all subcatchments"

The Black Box Foundation

      

      

        
Welcome. You're about to understand what's actually happening inside the EPA SWMM5 runoff engine — the same engine that runs in tens of thousands of consulting offices, water utilities, and university classrooms.

        
Most people who use SWMM5 treat the runoff engine as a black box. You set Manning's n, pick Horton or Green-Ampt, choose a design storm, click run. Numbers come out. They look reasonable. You move on.

2

        
The Physical Question Concept

      

      

        
A subcatchment is a piece of land. Rain falls on it. Water runs off. The engineering question is simple:

        

          
The Question

          
Given the rainfall, how much water comes out the bottom of this subcatchment, and when?

        

        
The answer matters. Pipes get sized from it. Detention ponds get sized from it. Flood elevations get computed from it. A 10% error in peak runoff might mean a pipe is one size too small, which means raw sewage backs up into someone's basement during a storm.

        
The honest answer is: we cannot solve this exactly. The real physics — millions of raindrops hitting heterogeneous surfaces, infiltrating through varying soil, flowing as thin films, collecting in tiny depressions, evaporating from warm asphalt — is computationally intractable.

        
So we abstract. SWMM5 uses the nonlinear reservoir model: treat the entire subcatchment as a shallow reservoir with one number — depth d. Rain raises the depth. Outflow lowers it. Infiltration and evaporation lower it.

        
dd/dt = i(t) − e(t) − f(t) − q(d)

        
That's the heart of it. i = rainfall, e = evaporation, f = infiltration, q(d) = outflow as a function of depth. Solving this ODE is what the entire runoff engine does.

      

    


    
    

      

        
3

        
The Three Surfaces You Never Knew About Reveal

      

      

        
Here is where SWMM5 does something subtle that almost no user notices.

        
A single nonlinear reservoir would be a lie. A parking lot has zero infiltration and almost no depression storage — rain hits, runs off, done. A lawn next to that parking lot has high infiltration capacity and significant depression storage — rain soaks in or fills small puddles before any runoff begins.

        
If you average these together, you get a soggy, slow-responding catchment that doesn't match either piece. The peak would be too low and arrive too late.

        
So SWMM5 splits each subcatchment into three surfaces that solve three independent ODEs in parallel:

        
    
          
  • A1 — Impervious without depression storage (% Imperv × % Zero-Imperv): The fast responder. Rain runs off the moment any depth exists.
    • 
          
  • A2 — Impervious with depression storage (% Imperv × (1 − % Zero-Imperv)): The delayed impervious responder. Must fill d_s first.
    • 
          
  • A3 — Pervious (1 − % Imperv): The slow responder. Subject to BOTH infiltration AND depression storage. May produce zero runoff for small storms.
    • 
        

        
This three-surface structure is invisible in every commercial SWMM wrapper — InfoSWMM, ICM, PCSWMM, XPSWMM. This app shows it happening: three separate hydrographs, three separate depth traces, three separate solver sub-step counts, all updating simultaneously.

        

          
▸ Try it:

          
Switch to the Simulator tab and look at the cross-section. You'll see three colored zones representing A1, A2, and A3. Watch how rainfall affects each one differently when you press Play.

          
        

      

    


    
    

      

        
4

        
Manning's Equation Where It Doesn't Belong Hidden Assumption

      

      

        
The outflow function q(d) in the nonlinear reservoir uses Manning's equation:

        
q = W · (1.49/n) · (d − d_s)5/3 · √S

        
This is a strange choice if you think about it. Manning's equation was derived for uniform open-channel flow — a long, straight pipe or river with steady flow. Subcatchment overland flow is none of those things. It's a thin sheet of water spreading across miles of non-uniform surface during an unsteady thunderstorm.

        
But the deep insight is that Manning's equation captures the right functional shape: depth raised to the 5/3 power means doubling the depth more than triples the outflow. This is what makes the reservoir "nonlinear" and makes the equation hard to solve.

        

          ⚠️ Common Modeler Confusion
          The W in Manning's equation is NOT the physical width of your subcatchment. It's a calibration parameter representing where overland flow finally concentrates. Rule of thumb: W ≈ Area / longest flow path.
        

        
The (d − d_s) term makes depression storage explicit. Below d_s, outflow is forced to zero. Above d_s, outflow turns on — sharply, because (5/3) applied to a small positive number changes rapidly. This non-smooth transition is exactly why the ODE is hard to solve numerically. We'll see why next.

      

    


    
    

      

        
5

        
Why Cash-Karp RK5? Numerical Methods

      

      

        
You can't write down d(t) in closed form. So you discretize: pick a small time step h, advance step by step.

        
The simplest approach is Euler's method: d(t+h) ≈ d(t) + h·f(t,d(t)). It's first-order accurate and terrible — error proportional to h, errors compound, sharp transitions cause catastrophic failure.

        
Classical RK4 evaluates f at four points within the step and combines them with carefully chosen weights to cancel error terms. Fourth-order accurate — error proportional to h⁴. This is the workhorse of most ODE solvers.

        
Cash-Karp RK5 takes this further. It evaluates f at six carefully placed points and combines them in two different ways to produce both a 5th-order solution y₅ AND a 4th-order solution y₄ from the same six function evaluations.

        
Why would you want both? Because:

        
|y₅ − y₄| ≈ truncation error of y₄

        
The difference between the two embedded solutions tells you how big your numerical error is. And once you know that, you can do the magic trick: adapt the step size.

        
    
          
  • Compute y₄ and y₅ with step h
    • 
          
  • If err > tolerance: reject the step, shrink h, retry
    • 
          
  • If err < tolerance: accept the step, advance, grow h for next time
    • 
        

        
Tiny steps when the ODE is hard. Large steps when it's easy. The solver becomes self-tuning.

6

        
Reading the Sub-Step Fingerprint Insight

      

      

        
Now we connect the math to physical insight. The chart labeled "Infiltration & RK5 Sub-Steps" in the Simulator tab plots how many times the adaptive solver subdivided each routing time step.

        
A height of 1 means the solver got it right on the first try. A height of 8 means it had to subdivide that interval into eight smaller pieces.

        
You will see three patterns:

        
    
          
  • A1 (impervious, no DS) — most sub-steps during rising and falling limbs. No buffer means depth changes sharply, which means (5/3) outflow varies rapidly.
    • 
          
  • A2 (impervious, with DS) — sub-steps spike when depth crosses the depression storage threshold. Below d_s, ODE is trivial. Above, outflow turns on. The transition is non-smooth and RK5 struggles at kinks.
    • 
          
  • A3 (pervious) — fewest sub-steps during the storm peak. Counterintuitive at first, but infiltration and depression storage act as dampers. The most physical surface is the easiest to integrate.
    • 
        

        

          
Key Insight

          
The sub-step chart is a fingerprint of where the storm is hardest numerically. You cannot get this insight from looking at hydrographs alone. It requires watching the solver work.

        

      

    


    
    

      

        
7

        
Three Infiltration Models Choice

      

      

        
Pervious surfaces lose water to infiltration. The app implements three different models, and the choice matters more than most modelers admit.

        
Horton is empirical:

        
f(t) = f∞ + (f₀ − f∞) · exp(−k·t)

        
Infiltration starts at maximum rate f₀ (dry soil), decays exponentially to minimum f∞ (saturated), with rate constant k. The math is clean. The physics is wrong — soil infiltration depends on how much water has already infiltrated, not on time directly. But Horton is computationally cheap and engineers have decades of calibrated values, so it dominates.

        
Green-Ampt is physically based:

        
f = K_s · (1 + Ψ·Δθ/F)

        
Models a sharp wetting front advancing into the soil. K_s = saturated conductivity, Ψ = suction head, Δθ = moisture deficit, F = cumulative infiltration. Physically correct. Iterative.

        
SCS Curve Number is engineering-empirical:

        
Q = (P − I_a)² / (P − I_a + S),  S = 1000/CN − 10

        
Not really an infiltration model — it's a runoff volume model retrofitted to per-step infiltration. Most-used model in US practice because the SCS published curve numbers for every soil-cover combination in the country.

Design Storms — Why They Differ Engineering

      

      

        
Before the engine matters, you need rainfall. The app has seven design storm patterns, each with engineering history.

        
    
          
  • SCS Type II (used east of the Rockies) has a vicious central burst delivering 70% of the storm in ~4 hours around hour 12. This central burst is what makes A1's RK5 sub-step count spike at hour 12.
    • 
          
  • SCS Type IA (Pacific Northwest) is much more uniform. Same total depth, dramatically different runoff response.
    • 
          
  • Chicago (Keifer-Chu, 1957) is the original "design storm" — parameterized peak position with rising and falling limbs. Used heavily in urban drainage.
    • 
          
  • Uniform is constant intensity. Useful for sanity checks.
    • 
          
  • Triangular linearly rises and falls. Teaching shape — easy to reason about.
    • 
        

        
The same total rainfall depth produces very different peak runoff depending on the temporal distribution. Try a 4-inch SCS Type II vs. a 4-inch Uniform — the peaks differ by a factor of 5.

Mass Balance — The Sanity Check QA/QC

      

      

        
Every fluid simulation must conserve mass. Water in equals water out plus water stored. The "continuity error" is the residual of this balance:

        
Continuity Error = Rain − Runoff − Infiltration − Evaporation − ΔStorage

        
A correctly running RK5 simulation should have continuity error well under 1%. If you see it growing above 5%, something is wrong — usually time step too coarse, or depression storage being added/removed inconsistently.

        
Engineers learn to check this number reflexively. A SWMM5 model with 3% continuity error is suspect. A model with 0.05% continuity error is trustworthy.

        
The Water Balance Summary on the Simulator tab shows runoff (blue), infiltration (gold), evaporation (purple), depression storage (indigo), and error (red) as fractions of total rainfall.

        

          📊 Reading the bar
          A well-behaved simulation has no visible red. If you see red, change the wet time step to a finer value (1 or 2 min) and run again. The error should shrink.
